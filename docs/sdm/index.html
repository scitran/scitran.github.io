Groups

Groups are a relatively loose concept in scitran. Even though projects are owned by groups, group membership itself does not grant any permissions on projects owned by that group. New projects, especially those created by the reaper, inherit their initial permissions from the respective group, but those permissions can be arbitrarily changed later. The SDM UI presents projects grouped by group, but does not take group membership into account for data presentation. Other presentation options might emphasize projects that are owned by groups that the user is a member of. For admin activities, such as creating a new project within a group or modifying group roles, membership is of course relevant.

Only the site admin (superuser) can create new groups.

Users do not need to be members of any groups. For our original use-case, which is to collect and present all data generated at an imaging research center, groups are generally a representation of the research labs using the center. Membership itself is managed within each lab.
